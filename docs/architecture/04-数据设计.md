# 数据设计

> 一句话定位：本文档回答「链上和前端各有什么数据、数据怎么流转」，受众是开发者。

## 链上数据模型

### MyERC20 合约状态

| 状态 | 类型 | 读方法 | 写方法 |
|---|---|---|---|
| 代币名称 | string | `name()` | — |
| 代币符号 | string | `symbol()` | — |
| 小数位 | uint8 | `decimals()` | — |
| 总供应量 | uint256 | `totalSupply()` | — |
| 账户余额 | mapping(address → uint256) | `balanceOf(account)` | `transfer`, `transferFrom` |
| 授权额度 | mapping(address → mapping(address → uint256)) | `allowance(owner, spender)` | `approve` |

### TokenBank 合约状态

| 状态 | 类型 | 读方法 | 写方法 |
|---|---|---|---|
| 底层代币 | address | `token()` | — |
| 用户存款 | mapping(address → uint256) | `balanceOf(user)` / `deposits(user)` | `deposit`, `withdraw` |

### NFTMarket 合约状态

| 状态 | 类型 | 读方法 | 写方法 |
|---|---|---|---|
| 支付代币 | address | `paymentToken()` | — |
| 下一上架 ID | uint256 | `nextListingId()` | — |
| 上架记录 | mapping(uint256 → Listing) | `listings(id)` | `list` 创建 / `cancelListing` 标记 isActive=false / `buyNFT` 标记 isActive=false |

**Listing 结构体**：

```solidity
struct Listing {
  address seller;        // 卖家
  address nftContract;   // NFT 合约地址
  uint256 tokenId;       // NFT tokenId
  uint256 price;         // 价格（paymentToken 计价）
  bool isActive;         // 是否活跃
}
```

### NFTMarket 事件

| 事件 | 参数 | 触发时机 |
|---|---|---|
| `NFTListed` | listingId, seller, nftContract, tokenId, price | `list` 成功 |
| `NFTSold` | listingId, buyer, seller, nftContract, tokenId, price | `buyNFT` 成功 |
| `NFTListingCancelled` | listingId | `cancelListing` 成功 |

前端通过 `useNFTMarketEvents` 订阅这三个事件，详见 [03-模块设计/Hooks与数据流](./03-模块设计/Hooks与数据流.md)。

### 关键不变量

**TokenBank**：
- `deposit(amount)` 要求 `allowance(user, tokenBank) >= amount`，否则 `safeTransferFrom` revert
- `withdraw(amount)` 要求 `balanceOf(user) >= amount`
- 存款后：ERC20 `balanceOf(user)` 减少、TokenBank `balanceOf(user)` 增加、ERC20 `allowance` 减少（transferFrom 消耗）
- 取款后：TokenBank `balanceOf(user)` 减少、ERC20 `balanceOf(user)` 回升

**NFTMarket**：
- `list` 要求卖家是 `nftContract.tokenId` 的 owner，且已 `setApprovalForAll(market, true)`
- `buyNFT` 要求 `allowance(buyer, market) >= price` 且 `allowance(buyer, market)` 足够支付
- `buyNFT` 要求 `listings[id].isActive == true`
- 购买后：卖家 ERC20 余额增加、买家 ERC20 余额减少、买家 allowance 减少、NFT 所有权转移、listing.isActive = false

## 前端数据状态

### 钱包状态（WalletContext）

| 状态 | 类型 | 来源 |
|---|---|---|
| `account` | `Address \| null` | `walletClient.requestAddresses()` |
| `chainId` | `number \| null` | `walletClient.getChainId()` |
| `isConnecting` | `boolean` | 本地 |
| `error` | `string \| null` | 捕获异常 |
| `walletClient` | `WalletClient \| null` | `getWalletClient(provider)` |
| `publicClient` | `PublicClient` | `getPublicClient()` 单例 |

### Hook 数据状态

**TokenBank**：

| Hook | `data` 类型 | 轮询 | 触发刷新的事件 |
|---|---|---|---|
| `useTokenMetadata` | `TokenMetadata \| null` | 否 | 无（不可变） |
| `useTokenBalance` | `bigint \| null` | 4s | approve, deposit, withdraw 后 |
| `useAllowance` | `bigint \| null` | 4s | approve, deposit 后 |
| `useDepositBalance` | `bigint \| null` | 4s | deposit, withdraw 后 |

**NFTMarket**：

| Hook | `data` 类型 | 机制 | 触发刷新的事件 |
|---|---|---|---|
| `useListings` | `ListingInfo[]` | 6s 轮询 + 手动 refetch | list / buy / cancel 后调 refresh() |
| `useNFTMarketEvents` | `MarketLog[]` | 历史拉取 + 2s watchEvent 订阅 | 链上 NFTListed/NFTSold/NFTListingCancelled |

```typescript
type TokenMetadata = {
  name: string;
  symbol: string;
  decimals: number;
};

type ListingInfo = {
  listingId: bigint;
  seller: Address;
  nftContract: Address;
  tokenId: bigint;
  price: bigint;
  isActive: boolean;
};

type MarketLog = {
  id: string;
  eventName: "NFTListed" | "NFTSold" | "NFTListingCancelled";
  blockNumber: bigint | null;
  txHash: string;
  listingId: bigint | null;
  timestamp: number;
  seller?: Address;
  buyer?: Address;
  nftContract?: Address;
  tokenId?: bigint;
  price?: bigint;
};
```

### 操作组件本地状态

| 状态 | 类型 | 说明 |

| `amount` | `string` | 用户输入（人类可读单位） |

## 数据流图

### TokenBank 读取流（页面加载）

```
tokenbank/page.tsx
  │
  ├─ useTokenMetadata(configOk)
  │    └─ publicClient.readContract → name/symbol/decimals
  │
  ├─ useTokenBalance(account, enabled)
  │    └─ publicClient.readContract → balanceOf
  │
  ├─ useAllowance(account, tokenBankAddress, enabled)
  │    └─ publicClient.readContract → allowance
  │
  └─ useDepositBalance(account, enabled)
       └─ publicClient.readContract → TokenBank.balanceOf

  数据通过 props 下发给 5 个卡片组件
```

### NFTMarket 读取流（页面加载）

```
nft-market/page.tsx
  │
  ├─ useTokenMetadata(!!tokenAddress)
  │    └─ publicClient.readContract → name/symbol/decimals（用于价格展示）
  │
  ├─ useListings(configOk)
  │    └─ publicClient.readContract → nextListingId
  │    └─ 循环 readContract → listings(0..nextId-1)
  │    └─ 过滤 isActive === true → ListingInfo[]
  │
  └─ useNFTMarketEvents(configOk)
       └─ publicClient.getContractEvents({ abi, fromBlock: 0n }) → 历史事件
       └─ publicClient.watchEvent({ events, pollingInterval: 2000 }) → 实时事件
       └─ 解码 + Set 去重 → MarketLog[]
       └─ 每条 console.log 打印

  数据通过 props 下发给 6 个 NFTMarket 组件
```

### TokenBank 写入流（以 deposit 为例）

```
DepositCard
  │
  ├─ safeParseTokenAmount(amount, decimals) → bigint
  ├─ 校验: amount > 0 && ≤ balance && ≤ allowance
  │
  ├─ walletClient.writeContract({ TokenBank.deposit(amount), chain })
  │    └─ MetaMask 弹窗 → 用户签名 → 广播到 anvil
  │
  ├─ publicClient.waitForTransactionReceipt({ hash })
  │    └─ 轮询 anvil 直到出块
  │
  └─ receipt.status === "success"
       └─ Promise.all([
            refresh.balance(),     // useTokenBalance.refetch
            refresh.deposit(),     // useDepositBalance.refetch
            refresh.allowance(),   // useAllowance.refetch
          ])
```

### NFTMarket 写入流（以 buyNFT 为例）

```
BuyCard
  │
  ├─ listingId 输入 → 找到 ListingInfo
  ├─ 检查 erc20 allowance(account, nftMarketAddress) ≥ price?
  │    └─ 不足 → 先 walletClient.writeContract({ erc20.approve(market, price) })
  │
  ├─ walletClient.writeContract({ NFTMarket.buyNFT(listingId), chain })
  │    └─ MetaMask 弹窗 → 用户签名 → 广播到 anvil
  │
  ├─ publicClient.waitForTransactionReceipt({ hash })
  │
  └─ receipt.status === "success"
       └─ Promise.all([
            refresh(),           // useListings.refetch → ListingsTable 更新
            fetchAllowance(),    // BuyCard 内部 allowance 状态更新
          ])

  // 同时 useNFTMarketEvents 的 watchEvent 在下一个轮询周期捕获 NFTSold
  // → EventLogCard 追加日志 + console.log 打印
```

### 金额转换流

```
用户输入 "100.5"
  → safeParseTokenAmount("100.5", 18)
  → parseUnits("100.5", 18)
  → 100500000000000000000n  (bigint)

链上返回 100500000000000000000n
  → formatTokenAmount(100500000000000000000n, 18, 6)
  → "100.5"
  → 显示 "100.5 MTK"
```
