# 接口契约

> 一句话定位：本文档回答「合约 ABI、Hook 返回类型、组件 Props 各是什么形状」，受众是对接合约或扩展组件的开发者。

## 合约 ABI

### erc20Abi（MyERC20）

| 类型 | 名称 | 签名 | 读写 |
|---|---|---|---|
| function | `name` | `() → string` | view |
| function | `symbol` | `() → string` | view |
| function | `decimals` | `() → uint8` | view |
| function | `balanceOf` | `(address account) → uint256` | view |
| function | `allowance` | `(address owner, address spender) → uint256` | view |
| function | `approve` | `(address spender, uint256 value) → bool` | nonpayable |
| event | `Approval` | `(address indexed owner, address indexed spender, uint256 value)` | — |
| event | `Transfer` | `(address indexed from, address indexed to, uint256 value)` | — |

### tokenBankAbi（TokenBank）

| 类型 | 名称 | 签名 | 读写 |
|---|---|---|---|
| function | `deposit` | `(uint256 _amount)` | nonpayable |
| function | `withdraw` | `(uint256 _amount)` | nonpayable |
| function | `balanceOf` | `(address _user) → uint256` | view |
| function | `deposits` | `(address) → uint256` | view |
| function | `token` | `() → address` | view |
| event | `Deposit` | `(address indexed user, uint256 amount)` | — |
| event | `Withdraw` | `(address indexed user, uint256 amount)` | — |
| error | `SafeERC20FailedOperation` | `(address token)` | — |

### erc721Abi（最小 ERC721，NFTMarket 用）

| 类型 | 名称 | 签名 | 读写 |
|---|---|---|---|
| function | `ownerOf` | `(uint256 tokenId) → address` | view |
| function | `approve` | `(address to, uint256 tokenId)` | nonpayable |
| function | `setApprovalForAll` | `(address operator, bool approved)` | nonpayable |
| function | `isApprovedForAll` | `(address owner, address operator) → bool` | view |
| function | `getApproved` | `(uint256 tokenId) → address` | view |
| event | `Approval` | `(address indexed owner, address indexed approved, uint256 indexed tokenId)` | — |
| event | `Transfer` | `(address indexed from, address indexed to, uint256 indexed tokenId)` | — |

### nftMarketAbi（NFTMarket）

| 类型 | 名称 | 签名 | 读写 |
|---|---|---|---|
| function | `paymentToken` | `() → address` | view |
| function | `nextListingId` | `() → uint256` | view |
| function | `listings` | `(uint256) → Listing` | view |
| function | `list` | `(address _nftContract, uint256 _tokenId, uint256 _price) → uint256` | nonpayable |
| function | `cancelListing` | `(uint256 _listingId)` | nonpayable |
| function | `buyNFT` | `(uint256 _listingId)` | nonpayable |
| function | `buyNFTWithCallback` | `(uint256 _listingId)` | nonpayable |
| event | `NFTListed` | `(uint256 indexed listingId, address indexed seller, address indexed nftContract, uint256 tokenId, uint256 price)` | — |
| event | `NFTSold` | `(uint256 indexed listingId, address indexed buyer, address indexed seller, address nftContract, uint256 tokenId, uint256 price)` | — |
| event | `NFTListingCancelled` | `(uint256 indexed listingId)` | — |

`listings` 返回的 `Listing` 是带命名 components 的 tuple，viem 解码为命名对象 `{ seller, nftContract, tokenId, price, isActive }`。

### nftMarketEvents（AbiEvent 数组）

仅含 `NFTListed`、`NFTSold`、`NFTListingCancelled` 三个事件，与 `nftMarketAbi` 中事件部分重复但独立导出。

**为什么单独导出**：viem 的 `watchEvent` 只接受 `event`（单个 AbiEvent）或 `events`（AbiEvent 数组），不接受完整 `abi`；而 `getContractEvents` 接受完整 `abi`。故拉历史用 `nftMarketAbi`，订阅实时用 `nftMarketEvents`。

### SimpleNft（测试合约，无前端 ABI 文件）

NFTMarket 测试用的最小 ERC721 合约（`contracts/src/SimpleNft.sol`），前端不直接导入其 ABI——`erc721Abi`（在 `nftMarketAbi.ts` 中）已覆盖前端所需的 ownerOf/approve/setApprovalForAll 等调用。测试脚本（`test/nftmarket/mint-nfts.mjs`）用 `parseAbi` 内联声明所需函数：

| 类型 | 名称 | 签名 | 读写 |
|---|---|---|---|
| function | `mint` | `(address to, uint256 tokenId)` | nonpayable |
| function | `ownerOf` | `(uint256 tokenId) → address` | view |
| function | `setApprovalForAll` | `(address operator, bool approved)` | nonpayable |
| function | `isApprovedForAll` | `(address owner, address operator) → bool` | view |
| function | `getApproved` | `(uint256 tokenId) → address` | view |

> SimpleNft 无 `safeTransferFrom` 回调校验、无 tokenURI、无 IERC721Metadata；`mint` 直接写 `_owners` 映射，不触发 Transfer 事件。仅用于 NFTMarket 上架/购买流程测试。

> 所有 ABI 以 `as const` 导出，viem 据此推断 `readContract` / `writeContract` 的参数和返回类型。

## EIP-1193 Provider 接口

```typescript
type EIP1193Provider = {
  request: (args: { method: string; params?: unknown[] }) => Promise<unknown>;
  on(event: "accountsChanged", listener: (accounts: Address[]) => void): void;
  on(event: "chainChanged", listener: (chainId: string) => void): void;
  removeListener(...): void;
};
```

来源：`window.ethereum`（MetaMask 注入）。

## WalletContext 契约

```typescript
type WalletState = {
  account: Address | null;
  chainId: number | null;
  isConnecting: boolean;
  error: string | null;
  walletClient: WalletClient | null;
  publicClient: PublicClient;
  connect: () => Promise<void>;
  disconnect: () => void;
};
```

`useWallet()` 在 `<WalletProvider>` 外调用会抛 `"useWallet 必须在 <WalletProvider> 内使用。"`。

## Hook 返回类型

### 统一形状

```typescript
type HookReturn<T> = {
  data: T | null;
  isLoading: boolean;
  error: Error | null;
  refetch: () => Promise<void>;
};
```

### 各 Hook 签名

**共享与 TokenBank**（轮询类，返回 `HookReturn<T>`）：

```typescript
useTokenMetadata(enabled?: boolean): HookReturn<TokenMetadata>                    // src/hooks/useTokenMetadata.ts
useTokenBalance(account: Address | null, enabled?: boolean): HookReturn<bigint>   // src/hooks/tokenbank/useTokenBalance.ts
useAllowance(account: Address | null, spender: Address | null, enabled?: boolean): HookReturn<bigint>  // src/hooks/tokenbank/useAllowance.ts
useDepositBalance(account: Address | null, enabled?: boolean): HookReturn<bigint> // src/hooks/tokenbank/useDepositBalance.ts
```

**NFTMarket**：

```typescript
// src/hooks/nftmarket/useListings.ts
useListings(enabled?: boolean): {
  data: ListingInfo[];
  isLoading: boolean;
  error: string | null;        // 注意是 string，不是 Error
  refetch: () => Promise<void>;
}

// src/hooks/nftmarket/useNFTMarketEvents.ts
useNFTMarketEvents(enabled?: boolean): {
  logs: MarketLog[];
  isWatching: boolean;
  error: string | null;
  clear: () => void;           // 清空日志 + 已知 ID 集合
}
```

### TokenMetadata

```typescript
type TokenMetadata = {
  name: string;
  symbol: string;
  decimals: number;
};
```

### ListingInfo（NFTMarket）

```typescript
type ListingInfo = {
  listingId: bigint;
  seller: Address;
  nftContract: Address;
  tokenId: bigint;
  price: bigint;
  isActive: boolean;
};
```

### MarketLog（NFTMarket 事件日志）

```typescript
type MarketLog = {
  id: string;                    // `${txHash}:${logIndex}` 去重键
  eventName: "NFTListed" | "NFTSold" | "NFTListingCancelled";
  blockNumber: bigint | null;
  txHash: string;
  listingId: bigint | null;
  timestamp: number;
  seller?: Address;              // NFTListed / NFTSold
  buyer?: Address;               // 仅 NFTSold
  nftContract?: Address;         // NFTListed / NFTSold
  tokenId?: bigint;              // NFTListed / NFTSold
  price?: bigint;                // NFTListed / NFTSold
};
```

## RefreshFns 契约

**TokenBank**（多函数包）：

```typescript
type RefreshFns = {
  balance: () => Promise<void>;
  allowance: () => Promise<void>;
  deposit: () => Promise<void>;
};
```

由 `tokenbank/page.tsx` 的 `useMemo` 打包，传入操作组件的 `refresh` prop。

**NFTMarket**（单函数）：

```typescript
type RefreshFn = () => Promise<void>;
```

由 `nft-market/page.tsx` 的 `useMemo(() => listings.refetch, [listings.refetch])` 生成，下发给 ListCard / BuyCard / CancelCard 的 `refresh` prop。

## writeContract 调用契约

所有操作组件的 `writeContract` 必须显式传 `chain` 参数：

```typescript
await walletClient.writeContract({
  account,           // 来自 useWallet()
  address,           // 合约地址，来自 config/tokenbank 或 config/nftmarket
  abi,               // 来自 contracts/tokenBankAbi 或 contracts/nftMarketAbi 或 contracts/erc20Abi
  functionName,      // 字符串字面量
  args,              // 参数数组
  chain,             // ← 必须显式传！来自 @/config/shared
});
```

> 原因：`getWalletClient` 返回的 `WalletClient` 泛型中 chain 为 `Chain | undefined`，viem 要求此时显式传 chain。所有操作组件统一从 `@/config/shared` 导入 `chain`。
