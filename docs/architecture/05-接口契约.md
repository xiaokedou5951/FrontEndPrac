# 接口契约

> 一句话定位：本文档回答「合约 ABI、Hook 返回类型、组件 Props 各是什么形状」，受众是对接合约或扩展组件的开发者。

## 合约 ABI

### erc20Abi（MyERC20）

| 类型 | 名称 | 签名 | 读写 |
|---|---|---|---|
| function | `name` | `() → string` | view |
| function | `symbol` | `() → string` | view |
| function | `decimals` | `() → uint8` | view |
| function | `balanceOf` | `(address account) → uint256` | view |
| function | `allowance` | `(address owner, address spender) → uint256` | view |
| function | `approve` | `(address spender, uint256 value) → bool` | nonpayable |
| event | `Approval` | `(address indexed owner, address indexed spender, uint256 value)` | — |
| event | `Transfer` | `(address indexed from, address indexed to, uint256 value)` | — |

### tokenBankAbi（TokenBank）

| 类型 | 名称 | 签名 | 读写 |
|---|---|---|---|
| function | `deposit` | `(uint256 _amount)` | nonpayable |
| function | `withdraw` | `(uint256 _amount)` | nonpayable |
| function | `balanceOf` | `(address _user) → uint256` | view |
| function | `deposits` | `(address) → uint256` | view |
| function | `token` | `() → address` | view |
| event | `Deposit` | `(address indexed user, uint256 amount)` | — |
| event | `Withdraw` | `(address indexed user, uint256 amount)` | — |
| error | `SafeERC20FailedOperation` | `(address token)` | — |

> ABI 以 `as const` 导出，viem 据此推断 `readContract` / `writeContract` 的参数和返回类型。

## EIP-1193 Provider 接口

```typescript
type EIP1193Provider = {
  request: (args: { method: string; params?: unknown[] }) => Promise<unknown>;
  on(event: "accountsChanged", listener: (accounts: Address[]) => void): void;
  on(event: "chainChanged", listener: (chainId: string) => void): void;
  removeListener(...): void;
};
```

来源：`window.ethereum`（MetaMask 注入）。

## WalletContext 契约

```typescript
type WalletState = {
  account: Address | null;
  chainId: number | null;
  isConnecting: boolean;
  error: string | null;
  walletClient: WalletClient | null;
  publicClient: PublicClient;
  connect: () => Promise<void>;
  disconnect: () => void;
};
```

`useWallet()` 在 `<WalletProvider>` 外调用会抛 `"useWallet 必须在 <WalletProvider> 内使用。"`。

## Hook 返回类型

### 统一形状

```typescript
type HookReturn<T> = {
  data: T | null;
  isLoading: boolean;
  error: Error | null;
  refetch: () => Promise<void>;
};
```

### 各 Hook 签名

```typescript
useTokenMetadata(enabled?: boolean): HookReturn<TokenMetadata>
useTokenBalance(account: Address | null, enabled?: boolean): HookReturn<bigint>
useAllowance(account: Address | null, spender: Address | null, enabled?: boolean): HookReturn<bigint>
useDepositBalance(account: Address | null, enabled?: boolean): HookReturn<bigint>
```

### TokenMetadata

```typescript
type TokenMetadata = {
  name: string;
  symbol: string;
  decimals: number;
};
```

## RefreshFns 契约

```typescript
type RefreshFns = {
  balance: () => Promise<void>;
  allowance: () => Promise<void>;
  deposit: () => Promise<void>;
};
```

由页面层 `useMemo` 打包，传入操作组件的 `refresh` prop。

## 组件 Props 契约

### 展示组件

```typescript
// TokenBalanceCard
{
  account: Address | null;
  balance: bigint | null;
  isLoading: boolean;
  error: Error | null;
  metadata: TokenMetadata | null;
}

// DepositBalanceCard
{
  account: Address | null;
  depositBalance: bigint | null;
  isLoading: boolean;
  error: Error | null;
  metadata: TokenMetadata | null;
}
```

### 操作组件

```typescript
// AllowanceCard
{ allowance: bigint | null; metadata: TokenMetadata | null; refresh: RefreshFns }

// DepositCard
{ balance: bigint | null; allowance: bigint | null; metadata: TokenMetadata | null; refresh: RefreshFns }

// WithdrawCard
{ depositBalance: bigint | null; metadata: TokenMetadata | null; refresh: RefreshFns }
```

### WalletBar

无 Props，内部调 `useWallet()`。

## writeContract 调用契约

所有操作组件的 `writeContract` 必须显式传 `chain` 参数：

```typescript
await walletClient.writeContract({
  account,           // 来自 useWallet()
  address,           // 合约地址，来自 config/contracts
  abi,               // 来自 contracts/abis
  functionName,      // 字符串字面量
  args,              // 参数数组
  chain,             // ← 必须显式传！来自 config/contracts
});
```

> 原因：`getWalletClient` 返回的 `WalletClient` 泛型中 chain 为 `Chain | undefined`，viem 要求此时显式传 chain。
