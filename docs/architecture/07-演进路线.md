## 技术债

### 高优先级

| 债 | 描述 | 影响 | 建议 |
|---|---|---|---|
| 轮询替代 | 3 个 Hook 各 4s 轮询，每分钟 ~45 次 RPC | 本地无影响，生产浪费 | 改用 `watchEvent` 订阅 Transfer/Deposit/Withdraw |
| 链 ID 校验 | 用户切到非 31337 链时仅更新显示，不提示 | 交易会失败但用户不知道 | 添加链 ID 不匹配提示 + 引导切换 |
| 地址一致性 | TokenBank 构造的 token 地址可能与 env 不匹配 | 存款 revert 且错误信息不直观 | 启动时调 `TokenBank.token()` 校验与 env 一致 |

### 中优先级

| 债 | 描述 | 建议 |
|---|---|---|
| 无交易历史 | 用户看不到自己的操作记录 | 添加交易日志面板（监听事件） |
| 无 loading 骨架 | 数据加载时仅显示"加载中..."文字 | 添加 skeleton 组件 |
| 无交易确认反馈 | 交易成功后无 toast 提示 | 添加 toast 通知 |
| AmountInput 无小数位限制 | 可输入超过 decimals 的小数位 | 根据 metadata.decimals 限制输入 |

### 低优先级

| 债 | 描述 | 建议 |
|---|---|---|
| 无单元测试 | 学习项目未写测试 | 核心 hook 和工具函数可加 vitest |
| 无 E2E | 无自动化验证 | 可加 Playwright |
| ABI 手写 | 仅含用到的函数，可能遗漏 | 如需完整 ABI 可引入 wagmi cli 生成 |
| 无深色模式 | Tailwind 4 支持 `dark:` | 非核心，按需添加 |

## 扩展计划

### 短期（下一个合约 Demo）

PRD 第 1 行：「会有各种类型的合约，其对应的页面资源应各自独立」。

扩展方式：

```
新增 src/app/<contract-name>/page.tsx
新增 src/components/<contract-name>/
新增/扩展 src/contracts/abis.ts
复用 src/context/WalletContext
复用 src/components/ui/
复用 src/lib/viem
```

### 中期

- 提取通用 hook 模式为 `useContractRead` / `useContractWrite`（当有 3+ 合约 Demo 时）
- 添加交易待确认队列 UI
- 支持 Sepolia 测试网部署

### 长期

- 如需多链切换，重新评估是否引入 wagmi
- 添加交易事件实时订阅面板
- 国际化（i18n）

## 风险

| 风险 | 可能性 | 影响 | 缓解 |
|---|---|---|---|
| anvil 重启后地址变化 | 高 | 前端连不上合约 | env 驱动，更新 .env.local 即可 |
| TokenBank token 地址不匹配 | 中 | 存款 revert | 文档提醒 + 启动校验（技术债） |
| MetaMask 版本变化 | 低 | API 不兼容 | EIP-1193 是标准，风险低 |
| React 19 破坏性变更 | 低 | 编译/运行错误 | tsc + 实际运行验证 |
