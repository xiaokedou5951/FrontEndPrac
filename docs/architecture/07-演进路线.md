## 技术债

### 高优先级

| 债 | 描述 | 影响 | 建议 |
|---|---|---|---|
| TokenBank 轮询替代 | 3 个 Hook 各 4s 轮询，每分钟 ~45 次 RPC | 本地无影响，生产浪费 | 仿 NFTMarket 改用 `watchEvent` 订阅 Deposit/Withdraw/Approval |
| 链 ID 校验 | 用户切到非 31337 链时仅更新显示，不提示 | 交易会失败但用户不知道 | 添加链 ID 不匹配提示 + 引导切换 |
| 地址一致性（运行时） | 部署时已由 `deploy-contracts.sh` 的 `cast call` 回读校验 `TokenBank.token()` / `NFTMarket.paymentToken()` 与 MyERC20 地址一致；但运行时若手动改过 `.env.local` 仍可能不一致 | 存款/购买 revert 且错误信息不直观 | 启动时调 `TokenBank.token()` / `NFTMarket.paymentToken()` 校验与 env 一致（部署时校验已落地，运行时校验未做） |

### 中优先级

| 债 | 描述 | 建议 |
|---|---|---|
| 无 loading 骨架 | 数据加载时仅显示"加载中..."文字 | 添加 skeleton 组件 |
| 无交易确认反馈 | 交易成功后仅文字提示，无 toast | 添加 toast 通知 |
| AmountInput 无小数位限制 | 可输入超过 decimals 的小数位 | 根据 metadata.decimals 限制输入 |
| NFTMarket 事件订阅无 WebSocket | watchEvent 用 HTTP 轮询 2s | 前端内已验证 WS 可用（`watch-events-ws.mjs` 跑通）；前端 hook 改用 `webSocket()` transport 即可，技术路径已铺通 |
| useListings 全量拉取 | nextListingId 大时遍历所有 listing，O(n) RPC | 改用分页或事件增量更新 |

### 低优先级

| 债 | 描述 | 建议 |
|---|---|---|
| 无单元测试 | 学习项目未写测试 | 核心 hook 和工具函数可加 vitest；NFTMarket 已有独立测试脚本（铸造 + 事件监听）用于手动验证 |
| 无 E2E | 无自动化验证 | 可加 Playwright；NFTMarket 测试脚本可作为 E2E fixture 基础 |
| ABI 手写 | 仅含用到的函数，可能遗漏 | 如需完整 ABI 可引入 wagmi cli 生成 |
| 无深色模式 | Tailwind 4 支持 `dark:` | 非核心，按需添加 |

## 扩展计划

### 短期（第三个合约 Demo）

PRD 第 1 行：「会有各种类型的合约，其对应的页面资源应各自独立」。

NFTMarket 已验证此模式可行，扩展方式标准化：

```
新增 src/contracts/<feature>Abi.ts
新增 src/config/<feature>.ts
新增 src/hooks/<feature>/
新增 src/components/<feature>/
新增 src/app/<feature>/page.tsx
复用 src/contracts/erc20Abi.ts、src/config/shared.ts、src/context/、src/components/ui/、shared/、src/lib/
```

详见 [ADR-0004](./decisions/0004-按功能拆分资源.md)。

### 中期

- 提取通用 hook 模式为 `useContractRead` / `useContractWrite`（当有 3+ 合约 Demo 时）
- 添加交易待确认队列 UI
- 支持 Sepolia 测试网部署（`deploy-contracts.sh` 已支持通过 `RPC_URL` / `PRIVATE_KEY` 环境变量覆盖部署到任意 EVM 链，缺 Sepolia 上的合约部署验证与 CI 流水线）
- TokenBank 改用事件订阅（消除轮询技术债）

### 长期

- 如需多链切换，重新评估是否引入 wagmi
- 国际化（i18n）
- 前端事件订阅改用 WebSocket transport（测试脚本已验证可行，前端 hook 待迁移）

## 风险

| 风险 | 可能性 | 影响 | 缓解 |
|---|---|---|---|
| anvil 重启后地址变化 | 高 | 前端连不上合约 | env 驱动，重跑 `scripts/deploy-contracts.sh` 一键重新部署并写回 `.env.local` |
| 三个合约地址不一致 | 中 | 存款/购买 revert | `deploy-contracts.sh` 按依赖顺序部署，自动把 MyERC20 地址传给 TokenBank / NFTMarket，并用 `cast call` 回读校验 |
| MetaMask 版本变化 | 低 | API 不兼容 | EIP-1193 是标准，风险低 |
| React 19 破坏性变更 | 低 | 编译/运行错误 | tsc + 实际运行验证 |
| watchEvent 轮询丢事件 | 低 | EventLogCard 漏日志 | 历史事件拉取兜底；生产改 WebSocket |
