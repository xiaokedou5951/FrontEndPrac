# ADR-0004：按功能拆分资源

- 状态：已接受
- 日期：2026-07-13
- 决策者：项目作者

## 背景

PRD 明确要求：「会有各种类型的合约，其对应的页面资源应各自独立」。

TokenBank 阶段，所有资源集中在单文件中：

```
contracts/abis.ts          # erc20Abi + tokenBankAbi 混在一起
config/contracts.ts        # tokenAddress + tokenBankAddress + configOk + chain 混在一起
hooks/useAllowance.ts      # 平铺在 hooks/ 根目录
hooks/useDepositBalance.ts
hooks/useTokenBalance.ts
components/tokenbank/WalletBar.tsx  # TokenBank 专属，但实际可复用
```

新增 NFTMarket 功能后，如果继续往这些单文件里追加：

1. **`abis.ts` 膨胀**：erc20Abi + tokenBankAbi + erc721Abi + nftMarketAbi + nftMarketEvents 全堆一个文件，NFTMarket 改动会影响 TokenBank 的类型推断
2. **`contracts.ts` 命名冲突**：两功能都需要 `configOk` / `configError`，被迫加前缀 `nftMarketConfigOk`，使用方写起来冗长
3. **`hooks/` 根目录混乱**：5+ 个 hook 平铺，无法一眼看出哪个属于哪个功能
4. **`WalletBar` 误归类**：放在 `components/tokenbank/`，但 NFTMarket 也要用，跨功能引用造成假依赖
5. **耦合蔓延**：改 NFTMarket 的 ABI 时，TokenBank 的文件被触碰，git diff 噪音大，回归风险高

## 决策

将所有资源按功能维度拆分为三组目录：`shared/`、`tokenbank/`、`nftmarket/`。

### 拆分粒度

| 资源类型 | shared/ | tokenbank/ | nftmarket/ |
|---|---|---|---|
| ABI | `contracts/erc20Abi.ts` | `contracts/tokenBankAbi.ts` | `contracts/nftMarketAbi.ts`（含 erc721Abi、nftMarketEvents） |
| 配置 | `config/shared.ts`（rpcUrl、chain、tokenAddress） | `config/tokenbank.ts`（tokenBankAddress、configOk、configError） | `config/nftmarket.ts`（nftMarketAddress、configOk、configError） |
| Hooks | `hooks/useTokenMetadata.ts` | `hooks/tokenbank/`（3 个） | `hooks/nftmarket/`（2 个） |
| 组件 | `components/shared/WalletBar`、`components/ui/` | `components/tokenbank/` | `components/nftmarket/` |

### 共享判定规则

什么资源放 `shared/`：

1. **被两个及以上功能引用**：如 `erc20Abi`（TokenBank 存取款 + NFTMarket 支付代币都用）、`tokenAddress`、`chain`
2. **功能无关的基础设施**：如 `WalletContext`、`lib/viem`、UI 基础组件（Button/Card/AmountInput/AddressInput）
3. **跨功能复用的业务组件**：如 `WalletBar`（连接钱包，两功能都要）

什么资源留在功能目录：

1. **功能专属 ABI**：`tokenBankAbi`、`nftMarketAbi`、`erc721Abi`
2. **功能专属地址与校验**：`tokenBankAddress`、`nftMarketAddress` 及各自的 `configOk`/`configError`
3. **功能专属 hooks**：`useDepositBalance`、`useListings` 等
4. **功能专属组件**：`DepositCard`、`BuyCard` 等

### 命名约定

- 两个功能模块都导出 `configOk` / `configError` **同名**，避免 `nftMarketConfigOk` 这种前缀冗余。使用方按需从对应功能模块导入，不冲突。
- ABI 文件以功能名命名：`tokenBankAbi.ts`、`nftMarketAbi.ts`；共享的以代币标准命名：`erc20Abi.ts`。
- hooks 子目录名与功能名一致：`hooks/tokenbank/`、`hooks/nftmarket/`。

## 备选方案

### 方案 A：继续单文件，加前缀区分

`abis.ts` 里加 `nftMarketAbi`，`contracts.ts` 里加 `nftMarketConfigOk`。

**为什么不选**：
- 单文件持续膨胀，可维护性下降
- 前缀命名冗长，使用方代码噪音大
- 改一个功能触碰所有功能的文件，git diff 难审
- 违反 PRD「页面资源应各自独立」的明确要求

### 方案 B：按资源类型拆分，不按功能

`contracts/abis/erc20.ts`、`contracts/abis/tokenBank.ts`、`contracts/abis/nftMarket.ts`，config 和 hooks 同理。

**评价**：
- 比 A 好，但仍未体现「功能独立」的语义
- 删除某功能时要在多个资源类型目录下分别找文件，不直观
- 难以一眼看出某功能涉及哪些资源

### 方案 C：完全功能化目录

```
src/features/
├── tokenbank/
│   ├── abi.ts
│   ├── config.ts
│   ├── hooks/
│   └── components/
├── nftmarket/
│   ├── abi.ts
│   ├── config.ts
│   ├── hooks/
│   └── components/
└── shared/
    ├── erc20Abi.ts
    ├── config.ts
    └── ...
```

**为什么不选**：
- 改动幅度过大，需移动所有现有文件
- 与 Next.js 约定的 `app/`、`components/`、`hooks/`、`lib/` 顶层目录习惯不符
- 当前两功能规模小，按资源类型 + 功能子目录的折中已足够清晰
- 如未来功能增至 5+，可再考虑此方案

## 后果

**正面**：
- 功能间彻底解耦：`tokenbank/` 不引用 `nftmarket/`，反之亦然
- 改动隔离：修改 NFTMarket ABI 只触碰 `contracts/nftMarketAbi.ts`，TokenBank 文件零改动
- 删除某功能简单：删掉对应目录即可，shared 不受影响（除非该功能是 shared 的唯一消费者，此时应重新评估 shared 归属）
- 新功能扩展路径清晰：照 `shared/tokenbank/nftmarket` 三组模式新增一组
- `configOk` / `configError` 同名导出，使用方代码简洁

**负面**：
- 文件数量增加：3 个 ABI 文件 + 3 个 config 文件 + hooks 子目录，初次阅读需建立映射
- 共享判定需权衡：某资源从「仅一个功能用」变为「两个功能用」时，需手动上浮到 `shared/`（如未来 NFTMarket 也需要 `useDepositBalance` 时）
- 导入路径稍长：`@/hooks/tokenbank/useAllowance` 比 `@/hooks/useAllowance` 长

**关键约束**：
- 功能目录间**禁止横向依赖**：`components/tokenbank/` 不得 import `@/components/nftmarket/` 或 `@/hooks/nftmarket/`，反之亦然。需共享时上浮到 `shared/`。
- `shared/` 不得反向依赖功能目录：`shared/` 是底层，不能引用 `tokenbank/` 或 `nftmarket/`。
- 依赖矩阵见 [03-模块设计/README](../03-模块设计/README.md)。

## 迁移过程

从单文件结构迁移到三组目录的步骤：

1. 创建新文件：`erc20Abi.ts`、`tokenBankAbi.ts`、`nftMarketAbi.ts`、`shared.ts`、`tokenbank.ts`、`nftmarket.ts`、`hooks/tokenbank/*`、`hooks/nftmarket/*`、`components/shared/WalletBar.tsx`
2. 更新所有使用方的 import 路径（组件、页面、lib）
3. 重命名 `nftMarketConfigOk` / `nftMarketConfigError` → `configOk` / `configError`（与 TokenBank 对齐）
4. 删除旧文件：`abis.ts`、`contracts.ts`、`hooks/useXxx.ts`（5 个）、`components/tokenbank/WalletBar.tsx`
5. `npx tsc --noEmit` + `npm run build` 验证
