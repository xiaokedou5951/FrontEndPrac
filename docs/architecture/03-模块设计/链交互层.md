# 链交互层

> 一句话定位：本文档回答「钱包连接、链客户端、合约配置如何管理」，受众是开发者。

涵盖模块：`config/contracts.ts`、`contracts/abis.ts`、`lib/viem.ts`、`context/WalletContext.tsx`

## config/contracts.ts — 配置层

### 职责

从 `NEXT_PUBLIC_*` 环境变量读取合约地址和链配置，导出校验后的常量。

### 导出

| 导出名 | 类型 | 说明 |
|---|---|---|
| `rpcUrl` | `string` | RPC 端点，默认 `http://127.0.0.1:8545` |
| `chain` | `Chain` | 链定义；ID 31337 用 viem 内置 `foundry`，否则 `defineChain` 自建 |
| `tokenAddress` | `Address \| null` | MyERC20 地址，env 缺失或格式错则 `null` |
| `tokenBankAddress` | `Address \| null` | TokenBank 地址 |
| `configOk` | `boolean` | 两个地址都非 null 时为 `true` |
| `configError` | `string \| null` | 缺失时的中文提示 |

### 环境变量

| 变量 | 必填 | 默认值 |
|---|---|---|
| `NEXT_PUBLIC_TOKEN_ADDRESS` | 是 | — |
| `NEXT_PUBLIC_TOKENBANK_ADDRESS` | 是 | — |
| `NEXT_PUBLIC_RPC_URL` | 否 | `http://127.0.0.1:8545` |
| `NEXT_PUBLIC_CHAIN_ID` | 否 | `31337` |
| `NEXT_PUBLIC_CHAIN_NAME` | 否 | `Custom Chain`（仅非 31337 时生效） |

## contracts/abis.ts — ABI 层

### 职责

手写精简 ABI（仅用到的函数），用 `as const` 保证 TypeScript 类型推断。

### 导出

**`erc20Abi`**：name, symbol, decimals, balanceOf, allowance, approve + Approval/Transfer 事件

**`tokenBankAbi`**：deposit, withdraw, balanceOf, deposits, token + Deposit/Withdraw 事件 + SafeERC20FailedOperation error

> 不引入 wagmi cli 等自动生成工具，保持简单并利于学习。

## lib/viem.ts — 客户端工厂

### 职责

创建 viem 客户端实例 + 提供 Token 金额格式化工具。

### 导出

| 函数 | 签名 | 说明 |
|---|---|---|
| `getPublicClient()` | `→ PublicClient` | 单例，`http(rpcUrl)` 传输 |
| `getWalletClient(ethereum)` | `→ WalletClient` | 每次新建，`custom(ethereum)` 传输 |
| `formatTokenAmount(value, decimals, maxFrac?)` | `→ string` | 格式化 + 可选截断小数位 |
| `parseTokenAmount(value, decimals)` | `→ bigint` | 解析人类输入为 wei |
| `safeParseTokenAmount(value, decimals)` | `→ bigint` | 解析失败返回 `0n` |

### 设计要点

- **publicClient 单例**：`publicClientInstance` 模块级缓存，避免重复创建
- **walletClient 非单例**：依赖 `window.ethereum`，每次连接时新建
- **safeParseTokenAmount**：处理用户输入边界（过多小数位等），操作卡片用它做安全校验

## context/WalletContext.tsx — 钱包状态层

### 职责

管理钱包连接生命周期，暴露 walletClient 和 publicClient 给全树。

### 状态

```typescript
type WalletState = {
  account: Address | null;       // 当前账户
  chainId: number | null;        // 当前链 ID
  isConnecting: boolean;         // 连接中
  error: string | null;          // 连接错误信息
  walletClient: WalletClient | null;  // 连接后才有
  publicClient: PublicClient;    // 始终可用（单例）
  connect: () => Promise<void>;
  disconnect: () => void;
};
```

### 连接流程

```
connect()
  → 检测 window.ethereum（无则设 error）
  → getWalletClient(provider)
  → Promise.all([requestAddresses(), getChainId()])
  → setAccount / setChainId / setWalletClient
  → attachListeners(accountsChanged, chainChanged)
  → localStorage.setItem("tokenbank:walletConnected", "1")
```

### 静默重连

首次挂载时检查 `localStorage`：
- 有标记 + provider 存在 → `getAddresses()`（不弹窗）→ `getChainId()` → 恢复状态
- 失败 → 清除标记

### 事件监听

| 事件 | 处理 |
|---|---|
| `accountsChanged` | 空数组 → 断开；非空 → 更新 account |
| `chainChanged` | 更新 chainId |

> disconnect 不调用 wallet 方法（MetaMask 无断开 API），仅清空本地状态 + 移除监听 + 清除 localStorage。

### 稳定性

- `publicClient` 用 `useMemo` 稳定（只创建一次）
- context value 用 `useMemo` 稳定（依赖所有 state 和方法）
- `connect` / `disconnect` 用 `useCallback` 稳定
