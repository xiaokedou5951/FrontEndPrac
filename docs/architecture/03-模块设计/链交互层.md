# 链交互层

> 一句话定位：本文档回答「钱包连接、链客户端、合约配置如何管理」，受众是开发者。

涵盖模块：`config/shared.ts`、`config/tokenbank.ts`、`config/nftmarket.ts`、`contracts/erc20Abi.ts`、`contracts/tokenBankAbi.ts`、`contracts/nftMarketAbi.ts`、`lib/viem.ts`、`context/WalletContext.tsx`

## config/ — 配置层（按功能拆分）

### 职责

从 `NEXT_PUBLIC_*` 环境变量读取合约地址和链配置，导出校验后的常量。三份文件按功能与共享维度拆分。

### config/shared.ts（共享）

TokenBank 与 NFTMarket 都需要的底层配置。

| 导出名 | 类型 | 说明 |
|---|---|---|
| `rpcUrl` | `string` | RPC 端点，默认 `http://127.0.0.1:8545` |
| `chain` | `Chain` | 链定义；ID 31337 用 viem 内置 `foundry`，否则 `defineChain` 自建 |
| `tokenAddress` | `Address \| null` | MyERC20 地址（两功能共用：TokenBank 存取款 + NFTMarket 支付代币），env 缺失或格式错则 `null` |

### config/tokenbank.ts（TokenBank 专属）

从 `./shared` 导入 `tokenAddress`，附加 TokenBank 地址与校验。

| 导出名 | 类型 | 说明 |
|---|---|---|
| `tokenBankAddress` | `Address \| null` | TokenBank 地址 |
| `configOk` | `boolean` | `tokenAddress` 与 `tokenBankAddress` 都非 null 时为 `true` |
| `configError` | `string \| null` | 缺失时的中文提示 |

### config/nftmarket.ts（NFTMarket 专属）

独立于 `shared.ts`（NFTMarket 不依赖 tokenAddress 做配置校验，paymentToken 在合约层绑定）。

| 导出名 | 类型 | 说明 |
|---|---|---|
| `nftMarketAddress` | `Address \| null` | NFTMarket 地址 |
| `configOk` | `boolean` | `nftMarketAddress` 非 null 时为 `true` |
| `configError` | `string \| null` | 缺失时的中文提示 |

> 两个功能模块都导出 `configOk` / `configError` 同名导出，避免使用方写 `nftMarketConfigOk` 这种带前缀的冗长名字；导入时按需从对应功能模块取。

### 环境变量

| 变量 | 必填 | 默认值 | 用于 |
|---|---|---|---|
| `NEXT_PUBLIC_TOKEN_ADDRESS` | 是 | — | shared.ts（两功能共用） |
| `NEXT_PUBLIC_TOKENBANK_ADDRESS` | 是 | — | tokenbank.ts |
| `NEXT_PUBLIC_NFT_MARKET_ADDRESS` | 是 | — | nftmarket.ts |
| `NEXT_PUBLIC_RPC_URL` | 否 | `http://127.0.0.1:8545` | shared.ts |
| `NEXT_PUBLIC_CHAIN_ID` | 否 | `31337` | shared.ts |
| `NEXT_PUBLIC_CHAIN_NAME` | 否 | `Custom Chain`（仅非 31337 时生效） | shared.ts |

## contracts/ — ABI 层（按功能拆分）

### 职责

手写精简 ABI（仅用到的函数），用 `as const` 保证 TypeScript 类型推断。三份文件按功能与共享维度拆分。

### contracts/erc20Abi.ts（共享）

**`erc20Abi`**：name, symbol, decimals, balanceOf, allowance, approve + Approval/Transfer 事件

TokenBank 与 NFTMarket 共用：TokenBank 用它查/授权代币，NFTMarket 用它查 paymentToken 余额与授权市场额度。

### contracts/tokenBankAbi.ts（TokenBank 专属）

**`tokenBankAbi`**：deposit, withdraw, balanceOf, deposits, token + Deposit/Withdraw 事件 + SafeERC20FailedOperation error

### contracts/nftMarketAbi.ts（NFTMarket 专属）

| 导出名 | 说明 |
|---|---|
| `erc721Abi` | 最小 ERC721 ABI（ownerOf, approve, setApprovalForAll, isApprovedForAll, getApproved + Approval/Transfer 事件），用于授权和查询所有权 |
| `nftMarketAbi` | NFTMarket 合约 ABI（paymentToken, nextListingId, listings, list, cancelListing, buyNFT, buyNFTWithCallback + NFTListed/NFTSold/NFTListingCancelled 事件） |
| `nftMarketEvents` | 仅含三个事件的 AbiEvent 数组，专门给 `watchEvent` 用（`watchEvent` 不接受完整 `abi`，只接受 `event`/`events`） |

> `nftMarketEvents` 单独导出是 viem API 的硬约束，详见 [Hooks与数据流](./Hooks与数据流.md) 的「事件订阅机制」一节。

> 不引入 wagmi cli 等自动生成工具，保持简单并利于学习。

## lib/viem.ts — 客户端工厂

### 职责

创建 viem 客户端实例 + 提供 Token 金额格式化工具。从 `@/config/shared` 导入 `chain` 和 `rpcUrl`。

### 导出

| 函数 | 签名 | 说明 |
|---|---|---|
| `getPublicClient()` | `→ PublicClient` | 单例，`http(rpcUrl)` 传输 |
| `getWalletClient(ethereum)` | `→ WalletClient` | 每次新建，`custom(ethereum)` 传输 |
| `formatTokenAmount(value, decimals, maxFrac?)` | `→ string` | 格式化 + 可选截断小数位 |
| `parseTokenAmount(value, decimals)` | `→ bigint` | 解析人类输入为 wei |
| `safeParseTokenAmount(value, decimals)` | `→ bigint` | 解析失败返回 `0n` |

### 设计要点

- **publicClient 单例**：`publicClientInstance` 模块级缓存，避免重复创建
- **walletClient 非单例**：依赖 `window.ethereum`，每次连接时新建
- **safeParseTokenAmount**：处理用户输入边界（过多小数位等），操作卡片用它做安全校验

## context/WalletContext.tsx — 钱包状态层

### 职责

管理钱包连接生命周期，暴露 walletClient 和 publicClient 给全树。从 `@/config/shared` 导入 `chain`。

### 状态

```typescript
type WalletState = {
  account: Address | null;       // 当前账户
  chainId: number | null;        // 当前链 ID
  isConnecting: boolean;         // 连接中
  error: string | null;          // 连接错误信息
  walletClient: WalletClient | null;  // 连接后才有
  publicClient: PublicClient;    // 始终可用（单例）
  connect: () => Promise<void>;
  disconnect: () => void;
};
```

### 连接流程

```
connect()
  → 检测 window.ethereum（无则设 error）
  → getWalletClient(provider)
  → Promise.all([requestAddresses(), getChainId()])
  → setAccount / setChainId / setWalletClient
  → attachListeners(accountsChanged, chainChanged)
  → localStorage.setItem("tokenbank:walletConnected", "1")
```

### 静默重连

首次挂载时检查 `localStorage`：
- 有标记 + provider 存在 → `getAddresses()`（不弹窗）→ `getChainId()` → 恢复状态
- 失败 → 清除标记

### 事件监听

| 事件 | 处理 |
|---|---|
| `accountsChanged` | 空数组 → 断开；非空 → 更新 account |
| `chainChanged` | 更新 chainId |

> disconnect 不调用 wallet 方法（MetaMask 无断开 API），仅清空本地状态 + 移除监听 + 清除 localStorage。

### 稳定性

- `publicClient` 用 `useMemo` 稳定（只创建一次）
- context value 用 `useMemo` 稳定（依赖所有 state 和方法）
- `connect` / `disconnect` 用 `useCallback` 稳定
