# 业务上下文

> 一句话定位：本文档回答「项目做什么、为谁做、核心使用场景是什么」，受众是所有人。

## 项目范围

构建一个**学习型**前端项目，通过浏览器钱包与已部署的 Solidity 合约交互，理解 ERC20 授权流程与 TokenBank 存取款机制。

PRD 原文（`docs/prd/TokenBank需求文档.txt`）：

> 搭建一个前端项目，目标是学习与链上合约交互，会有各种类型的合约，其对应的页面资源应各自独立。
> 技术栈：Next.js、TypeScript、Tailwind CSS 4、Viem v2

### 五条核心需求

| # | 需求 | 对应组件 |
|---|---|---|
| 1 | 连接钱包 | WalletBar |
| 2 | 显示 Token 余额 + 可存款 | TokenBalanceCard + DepositCard |
| 3 | 存款后显示存款金额 | DepositBalanceCard |
| 4 | 支持取款 | WithdrawCard |
| 5 | 授权后显示当前授权额度 | AllowanceCard |

### 不在范围内

- wagmi / RainbowKit 集成
- 多链切换 UI（仅支持配置的单一链）
- 交易历史、事件日志订阅
- 单元测试 / E2E
- 国际化、深色模式

## 核心使用场景

### 场景 1：首次连接

```
用户打开 /tokenbank → 看到"连接钱包"按钮 → 点击 → MetaMask 弹窗 → 授权
→ 顶栏显示截断地址 + Chain 31337 → 余额卡片加载
```

### 场景 2：授权 → 存款

```
用户在 AllowanceCard 输入金额 → 点"授权" → MetaMask 确认 → 授权额度更新
→ 在 DepositCard 输入金额 → 点"存款" → MetaMask 确认
→ Token 余额减少、存款余额增加
```

### 场景 3：取款

```
用户在 WithdrawCard 输入金额 → 点"取款" → MetaMask 确认
→ 存款余额减少、Token 余额回升
```

### 场景 4：配置缺失

```
用户未配置 .env.local → 打开 /tokenbank → 看到配置缺失提示
→ 不会崩溃、不会发起任何链上请求
```

## 合约背景

| 合约 | 角色 | 关键约束 |
|---|---|---|
| MyERC20 | 标准 ERC20 代币 | 构造时向部署者铸造 1,000,000 * 1e18 |
| TokenBank | 存款银行 | `deposit` 内部调 `safeTransferFrom`，**必须先 approve** |

> ⚠️ TokenBank 构造时传入的 token 地址可能与当前 MyERC20 部署地址不一致，重新部署后需更新 `.env.local`。
