# 业务上下文

> 一句话定位：本文档回答「项目做什么、为谁做、核心使用场景是什么」，受众是所有人。

## 项目范围

构建一个**学习型**前端项目，通过浏览器钱包与已部署的 Solidity 合约交互。当前包含两个独立功能：

- **TokenBank**：理解 ERC20 授权流程与存取款机制
- **NFTMarket**：理解 ERC721 授权、上架/购买/取消流程，并实时监听链上事件

PRD 原文（`docs/prd/TokenBank需求文档.txt`）：

> 搭建一个前端项目，目标是学习与链上合约交互，会有各种类型的合约，其对应的页面资源应各自独立。
> 技术栈：Next.js、TypeScript、Tailwind CSS 4、Viem v2

### 核心需求

**TokenBank**

| # | 需求 | 对应组件 |
|---|---|---|
| 1 | 连接钱包 | WalletBar（shared） |
| 2 | 显示 Token 余额 + 可存款 | TokenBalanceCard + DepositCard |
| 3 | 存款后显示存款金额 | DepositBalanceCard |
| 4 | 支持取款 | WithdrawCard |
| 5 | 授权后显示当前授权额度 | AllowanceCard |

**NFTMarket**

| # | 需求 | 对应组件 |
|---|---|---|
| 1 | 上架 NFT（指定合约、tokenId、价格） | ListCard |
| 2 | 授权 NFT 合约给市场 | ApproveNFTCard |
| 3 | 按 listingId 购买（含支付代币额度检查与一键授权） | BuyCard |
| 4 | 取消上架 | CancelCard |
| 5 | 查看活跃上架列表 | ListingsTable |
| 6 | 后台监听上架/买卖事件并打印日志 | EventLogCard + useNFTMarketEvents |

### 不在范围内

- wagmi / RainbowKit 集成
- 多链切换 UI（仅支持配置的单一链）
- 单元测试 / E2E（已有独立测试脚本用于手动验证，见 `viem-front/test/nftmarket/`）
- 国际化、深色模式

## 核心使用场景

### 场景 1：首次连接

```
用户打开 /tokenbank 或 /nft-market → 看到"连接钱包"按钮 → 点击 → MetaMask 弹窗 → 授权
→ 顶栏显示截断地址 + Chain 31337 → 余额/列表卡片加载
```

### 场景 2：TokenBank 授权 → 存款

```
用户在 AllowanceCard 输入金额 → 点"授权" → MetaMask 确认 → 授权额度更新
→ 在 DepositCard 输入金额 → 点"存款" → MetaMask 确认
→ Token 余额减少、存款余额增加
```

### 场景 3：TokenBank 取款

```
用户在 WithdrawCard 输入金额 → 点"取款" → MetaMask 确认
→ 存款余额减少、Token 余额回升
```

### 场景 4：NFTMarket 上架 → 购买

```
卖家在 ApproveNFTCard 授权市场 → 在 ListCard 填合约/tokenId/价格 → 上架
→ EventLogCard 打印 NFTListed 日志，ListingsTable 出现新行
→ 买家在 BuyCard 输入 listingId →（额度不足时一键 approve）→ 购买
→ EventLogCard 打印 NFTSold 日志，ListingsTable 移除该行
```

### 场景 5：配置缺失

```
用户未配置 .env.local → 打开 /tokenbank 或 /nft-market → 看到配置缺失提示
→ 不会崩溃、不会发起任何链上请求
```

## 合约背景

| 合约 | 角色 | 关键约束 |
|---|---|---|
| MyERC20 | 标准 ERC20 代币 | 构造时向部署者铸造 1,000,000 * 1e18；同时作为 NFTMarket 的 paymentToken |
| TokenBank | 存款银行 | `deposit` 内部调 `safeTransferFrom`，**必须先 approve** |
| NFTMarket | NFT 交易市场 | 上架需先 `setApprovalForAll` 给市场；购买用 paymentToken 转账，需先 approve 市场足够的代币额度 |
| SimpleNft | NFTMarket 测试用 ERC721 | 仅实现最小 ERC721 接口（mint/ownerOf/transferFrom/approve/setApprovalForAll），无 URI、无元数据；`mint(to, tokenId)` 直接写 `_owners` 映射，供测试脚本批量铸造后上架 |

> ⚠️ TokenBank 构造时传入的 token 地址可能与当前 MyERC20 部署地址不一致，重新部署后需更新 `.env.local`。NFTMarket 部署脚本复用 `TOKEN_ADDRESS` 作为 paymentToken，保证三者一致。SimpleNft 是独立的测试合约，不参与 TokenBank / NFTMarket 的构造依赖。
>
> 推荐使用 [`viem-front/scripts/deploy-contracts.sh`](../../viem-front/scripts/deploy-contracts.sh) 一键完成部署：脚本按依赖顺序（MyERC20 → TokenBank / NFTMarket → SimpleNft）部署，自动把 MyERC20 地址传给后续合约，部署完用 `cast call` 回读校验 `TokenBank.token()` 与 `NFTMarket.paymentToken()` 一致，再把 4 个地址写回 `.env.local`。
